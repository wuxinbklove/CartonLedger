# 纸箱对账记账程序

基于 C++、Qt6、SQLite 和 CMake 的跨平台桌面应用，用于录入纸箱对账数据，支持按计算公式自动计算单价或直接手动录入单价，并支持 CSV 与 xlsx 导出。

## 当前已实现

- 表格录入计算公式、送货日期、订单号、规格、数量、每平方单价、单价、总价、备注
- 订单号可为空，规格为必填
- 备注可为空，内容会随数据一起保存到本地数据库
- 规格只保留一列手工输入，格式支持 `长*宽` 或 `长*宽*高`
- 计算公式列位于最前面，并根据规格格式自动限制：`长*宽` 可用 C 或 手动，`长*宽*高` 可用 A、B 或 手动
- 新增行默认使用计算公式 A；当规格为 `长*宽` 且未命中历史记录时，默认切换为 C
- 规格输入支持自动联想历史规格，并在匹配到最近使用记录后自动带出数量、每平方单价和计算公式
- 单价列始终允许编辑；手动输入时尽量保留输入的小数位数，不再受每平方单价小数位影响
- 编辑单价后会按输入保留位数逐级判断是否符合 A、B、C 的计算结果；只要在某个保留位数下满足，就自动切换对应计算公式，否则切换为手动
- 列表始终按数据库中的手工顺序显示，新增行支持插入到当前选中行之后，也支持一次插入多行
- 表格支持直接从 Excel 粘贴制表符数据，也支持复制选中内容回贴到 Excel
- 自动计算公式下，单价和总价默认保留两位小数；手动模式下，单价和总价跟随手动输入的单价
- 数据保存到本地 SQLite
- 按日期区间和订单号筛选
- 导出当前列表为 CSV
- 导出当前列表为 xlsx，可直接用 Excel 或 WPS 打开
- 设置默认每平方单价，默认值为 2.15

## 公式

- A: `((长 + 宽 + 4) * (宽 + 高 + 2) * 2) * 每平方单价 / 10000`
- B: `((长 + 宽 + 4) * (宽 + 高 + 宽 + 2) * 2) * 每平方单价 / 10000`
- C: `长 * 宽 * 每平方单价 / 10000`
- 手动: 直接输入单价
- 总价: `单价 * 数量`

## 构建

```bash
cmake -S . -B build -DCMAKE_PREFIX_PATH=/usr/local
cmake --build build
ctest --test-dir build --output-on-failure
```

如果本机的 Qt 安装路径已经被 CMake 自动识别，可以省略 `-DCMAKE_PREFIX_PATH=/usr/local`。

## 运行

```bash
./build/src/carton_ledger
```
